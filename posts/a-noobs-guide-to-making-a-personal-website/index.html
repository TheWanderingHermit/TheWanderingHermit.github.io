A noob's guide to creating a personal website

2022-06-14
guide
github , hugo , terminal , publishing

So, you finally decide to build your own website. You fire up your favorite search engine and take a quick look around on how to do so and are bogged down by the number of options available. Some of those provide you ready made websites with awesome bells and whistles, but charge you copious amount of money. Others are more easy on your pocket but require skills that only a computer engineer possesses. After straining your eyes and wits for hours, you decide it is not worth the effort.

Trust me it isn’t. And it doesn’t have to be. You can go from knowing nothing about website development to publishing your own site in under 20 minutes using the following steps. And I promise it is going to be easy on your pocket too (depending on your taste in names).

Gather the tools you need

  1. Create a Github account Github provides ability to store and manage source code of computer programs. Since a website is also a computer program, Github serves as an excellent place to store it. We will later see that Github can make the website available on the internet as well (we will call this process publishing the website). To create a Github account, go to the website and signup. In the process of signing up, you will be prompted to choose a USERNAME. Choose it carefully as it will become a part of your website address
  2. Create a Github repository by following the steps listed in this document
  3. Use the simple steps outlined in this document to install git on your computer

Create and publish the work

  1. Follow this quick start to install Hugo and create a local website
  2. Push the locally created website to your Github repository By now, I trust you have built some understanding of terminal. You will need to fire it up for the next steps. Don’t worry if you are intimidated by it, I will guide you through all the commands you need. Also, for the following steps, I will assume that hugo has created your website in /path/website folder and your github repository address is https://www.github.com/USERNAME/REPOSITORY.git. Modify the steps based on the names you have choosen
    cd /path/website
    git init
    git add .
    git remote add origin https://www.github.com/USERNAME/REPOSITORY.git
    git branch -M main
    git push -u origin main

  1. Create a Github action to publish the website whenever it is updated We are almost there. In this step, we will look at how to automatically make the website available to the prying eyes of the world (publish) when it is updated in the Github repository

    Github actions are like shortcuts on your iphone. Their underscoring feature is that they trigger a response (or a series of responses) when they observe a users input. Here, we will utilize Github action to generate and deploy (response) a website when a user updates (input) it.

    1. Create a personal access token that will let your Github securely write to a repository. Grant it write access
    2. Use this personal access token to create an encrypted secret for your repository.
    3. Assuming the USERNAME is the user name of your Github account, create a public repository and name it USERNAME.github.io
    4. Create a workflow file (it directs Github actions) with the following absolute path /path/website/.github/workflows/gh-pages.yml. Note to replace /path/website with the absolute base path of your website
    5. Add the following content to the workflow file (you will need to change the contents according to the paths and addresses that apply to you)
        name: github pages

        on:
          push:
            branches:
              - main  # Set a branch to deploy
          pull_request:

        jobs:
          deploy:
            runs-on: ubuntu-20.04
            steps:
              - uses: actions/checkout@v2
                with:
                  submodules: true  # Fetch Hugo themes (true OR recursive)
                  fetch-depth: 0    # Fetch all history for .GitInfo and .Lastmod

              - name: Setup Hugo
                uses: peaceiris/actions-hugo@v2
                with:
                  hugo-version: 'latest'
                  extended: true

              - name: Build
                run: hugo --minify

              - name: Deploy
                uses: peaceiris/actions-gh-pages@v3
                if: github.ref == 'refs/heads/main'
                with:
                  external_repository: USERNAME/USERNAME.github.io
                  personal_token: ${{ secrets.[NAME OF THE SECRET YOU CREATED] }}
                  user_name:  [YOUR GITHUB USER NAME]
                  user_email: [YOUR GITHUB USER EMAIL]
                  publish_dir: ./public

  1. Commit and push the modifications to your Github account using the following commands

    cd /path/website
    git add .
    git commit -m "YOUR COMMIT MESSAGE"
    git push -u origin main

And viola! There you have it. Log on to https://www.USERNAME.github.io and you will see the contents of your website in full glory, free of cost! And every time you make a new post, you just have to push it to the Github repository using the above step and it will be available in your website as soon as the update takes hold.

Next Steps

You haven’t had to spend a penny on your website till now. Isn’t that something. Now you can play around with your site as much as you want. You can modify how it looks by using one of the available themes, or customize a part of your website using custom css. Don’t like the address of your website (frankly, I think USERNAME.github.io is cumbersome and unappealing as well), use a custom domain for your site. The possibilities only stop with your imagination (and with the ability of your computer)
